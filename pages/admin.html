What is an approval process â€” explain all components and a real use case ADMINâ–¼

An Approval Process is a declarative tool that automates a sequence of steps for approving a record. When a record meets certain criteria, it enters the approval process and must be approved (or rejected) by designated users before it moves forward.

  • Entry Criteria â€” conditions that trigger the process (e.g., Loan Amount > $500,000)
  • Approval Steps â€” each step defines who approves (user, queue, related field user) and what they can do
  • Approver â€” specific user, manager field, queue, or based on formula
  • Approval Actions â€” what happens on approval (field update, email, task, outbound message)
  • Rejection Actions â€” what happens on rejection
  • Recall Actions â€” what happens when submitter recalls
  • Initial Submission Actions â€” happen when record is submitted (before approval)

Real use case in Mortgage: Loans above $750,000 required a senior underwriter to approve before closing. We built a 2-step approval process: Step 1 â€” Branch Manager approval. Step 2 â€” Senior Underwriter approval. Both must approve before the loan status could move to Clear to Close. The approval locked the record from editing during review.

// Submitting a record to an approval process via Apex
Approval.ProcessSubmitRequest submitReq = new Approval.ProcessSubmitRequest();
submitReq.setObjectId(loan.Id);
submitReq.setComments('High-value loan requiring senior approval');
submitReq.setSubmitterId(UserInfo.getUserId());
submitReq.setProcessDefinitionNameOrId('High_Value_Loan_Approval'); // process name

Approval.ProcessResult result = Approval.process(submitReq);
System.debug('Submitted: ' + result.isSuccess() + ' Status: ' + result.getInstanceStatus());
What are Record Types â€” when do you use them vs picklist values? ADMINâ–¼

Record Types allow you to offer different business processes, page layouts, and picklist values for different users on the same object. A Record Type is assigned to a profile or permission set â€” users with that assignment see those page layouts and picklist options.

  • Use Record Types when: Different user groups need different page layouts, different picklist values, or different approval processes on the same object
  • Don't use for: Simple categorisation that doesn't affect UI or process â€” a picklist field is enough
  • Mortgage example: Loan object with Record Types: Conventional, FHA, VA, USDA. Each shows different fields on page layout, has different picklist values for loan programs, and can have different approval processes

What are Approval Processes and how do they interact with Apex?ADMIN▼

Approval Processes route records through multi-step approval chains with configurable entry criteria, approvers, and actions on approval/rejection.

// Submit for approval from Apex
Approval.ProcessSubmitRequest req = new Approval.ProcessSubmitRequest();
req.setComments('Submitting for underwriter review');
req.setObjectId(loan.Id);
req.setProcessDefinitionNameOrId('Loan_Approval_Process');
Approval.ProcessResult result = Approval.process(req);

// Approve programmatically (e.g., auto-approve high credit score)
Approval.ProcessWorkitemRequest approveReq = new Approval.ProcessWorkitemRequest();
approveReq.setComments('Auto-approved: score ' + loan.Credit_Score__c);
approveReq.setAction('Approve'); // or 'Reject'
approveReq.setWorkitemId(workItemId);
Approval.process(approveReq);
Approval and locking

When a record is pending approval it may be locked for editing. Check ProcessInstance.Status before allowing edits. Record save during approval still fires before/after triggers normally.

What are the key governor limits that admins must understand?ADMIN▼
  • SOQL queries: 100 sync / 200 async â€” complex reports can trigger Apex that hits this
  • DML statements: 150 per transaction â€” mass record operations can fire triggers multiple times
  • Storage limits: Data + file storage per org â€” monitor in Setup â†’ Company Information
  • API request limits: 24-hour rolling window â€” excessive integration calls deplete daily allocation
  • Scheduled jobs: Max 100 â€” orgs with many scheduled processes approach this limit
  • Concurrent API requests: Long-running API calls can time out external integrations
Admin monitoring

Setup â†’ Apex Jobs for batch/scheduled status. Setup â†’ Event Log Files for API usage patterns. Setup â†’ Company Information for storage. These are your first stops for performance investigations.
