FDE Prep: Agentforce Core Concepts
Master the foundations of Salesforce AI Agents, Atlas Reasoning, and the Trust Layer
What is Agentforce and how does it differ from Einstein Bots? CORE

Agentforce is an evolution from intent-based bots to **autonomous agents**. Unlike bots that follow strict "If-Then" trees, Agentforce uses the **Atlas Reasoning Engine** to determine which "Action" to take based on the user's goal.

  • Intent vs. Goal: Bots respond to pre-defined intents; Agents pursue high-level goals.
  • Actions: Agents can invoke Apex, Flows, and Prompt Templates autonomously.
  • Data: Agents use real-time Data Cloud data for grounding without training the model.
FDE Prep: Data Cloud Foundation
Understanding Ingestion, Modeling, Identity Resolution, and Calculated Insights
Explain the Data Cloud "Ingest to Action" flow. DATA CLOUD
  1. Ingest: Connect data sources (Salesforce, S3, Snowflake, Web/Mobile).
  2. Map: Map Data Source Objects (DSO) to Data Model Objects (DMO) using the CIM (Cloud Information Model).
  3. Unify: Use Identity Resolution rules to create a "Unified Individual".
  4. Insight: Create Calculated Insights (SQL-based) or Streaming Insights.
  5. Act: Trigger Flows, send to Marketing Cloud, or ground Agentforce prompts.
Atlas Reasoning & Trust Layer
The brain and the safety shield of Agentforce
How does the Atlas Reasoning Engine work? REASONING

Atlas works in a loop: **Query Analysis -> Plan Building -> Execution -> Result Evaluation**. It retrieves relevant metadata (Actions, Flows) and data (Data Cloud) to build a prompt, then interprets the LLM response to see if the goal is met or if more actions are needed.

FDE Prep: Einstein Trust Layer
Security, Privacy, and Compliance in the Age of Generative AI
What are the key features of the Einstein Trust Layer? TRUST

The Trust Layer sits between the Agent and the LLM. It ensures:

  • Dynamic Grounding: Pulls real data from Salesforce/Data Cloud to give the LLM context.
  • Data Masking: Replaces PII (email, phone, SSN) with placeholders before sending to the LLM.
  • Zero Retention: LLM providers (OpenAI, etc.) agree not to store or use data for training.
  • Toxicity Detection: Checks both input and output for offensive content.
  • Audit Trail: Logs every interaction for compliance.
